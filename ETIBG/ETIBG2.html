WHAT IS IT?

A demonstration of the logic behind MacArthur and Wilson’s equilibrium model of Island Biogeography. The model shows that the “equilibium” is in fact dynamic. There will be stochastic fluctuations around the equilibrium number of species.

HOW IT WORKS

The model is intiated with 14 species on the continent. This number is simply the number of clearly distinct colours that netlogo has. In each time step the individuals of the species move a random amount. Individuals that find themselves in the sea start “swimming” quickly in the direction they are facing. If they do not reach the island their energy is exhausted and they die. Those that do manage to colonise the island can begin to replicate and form a population using simple rules that remove individuals through crowding. Similar rules are applied to the continental population with a slight difference to allow for scaling. New individuals reaching the island may colonise, but lack of space leads to faster extinction (turnover) rates as time goes on.

HOW TO USE IT

Draw some islands before running the model, or nothing much will happen!

THINGS TO NOTICE

If colonisation is slow the first species to arrive may form large populations that lower the equilibrium number of species for a long time. This is quite an interesting insight from watching the simulations. Other features of the system are worth watching for.

THINGS TO TRY

As it stand the model cannot be altered by the user. Vector layers with different configurations could be loaded.

EXTENDING THE MODEL

Clearly lots more to do. But it works.

NETLOGO FEATURES

One issue that is unexpectedly difficult is graphing the dynamics of each species separately. This would be trivial in R and it may be simpler to use the RNetLogo interface. However this would lead to a setup overhead if the model were used for teaching.

CODE

turtles-own [ sp energy]
patches-own [gid]


to setup 
clear-all
reset-ticks
ask patches [ set pcolor blue set gid 0]
setup-continent
;;setup-island
end

to setup-continent
  ask patches with [pxcor < 0] [
   set pcolor green
   set gid 1
   if random 10 > 1 [
   sprout 1 [
   set shape "circle"
   set sp random 14
   set energy 28
   set color 5 + sp * 10
   ]]
]
end


to setup-island
  ask patches with [gid = 2][
  set pcolor green
  ]
end


to go
  move
  island-pop-dynamics
  continental-pop-dynamics
  tick
end

to move
  ask turtles [
    if [pcolor] of patch-here = green [
    rt random 360
    fd 0.1]
    if [pcolor] of patch-here = blue 
    [set energy energy - 1
    fd 2]
    if energy < 1 [
      die
      ask one-of patches with [gid = 1] [sprout 1]
  ]
  ]
end


to island-pop-dynamics
 ask turtles [
 if [gid] of patch-here = 2
 [if random 10 < 3 [hatch 1]
  fd 0.1
  if count turtles-here > 2 [die]
 ]]
end
 
 to continental-pop-dynamics
 ask turtles [
 if [gid] of patch-here = 1
 [if random 20 < 1 [hatch 1]
  fd 0.1
  if count turtles-here > 2 [die]
 ]]
   
end


to-report sp_on_island
 report length remove-duplicates [sp] of turtles-on  patches with [gid = 2 ]
end


to-report sp_on_continent
 report length remove-duplicates [sp] of turtles-on  patches with [gid = 1 ]
end

to draw-island
  if mouse-down?
  [ ask patch mouse-xcor mouse-ycor [ 
      set pcolor green
      set gid 2 
   ] ]
end

to draw-sea
  if mouse-down?
  [ ask patch mouse-xcor mouse-ycor [ 
      set pcolor blue
      set gid 0 
   ] ]
 
end

to draw-continent
  if mouse-down?
  [ ask patch mouse-xcor mouse-ycor [ 
      set pcolor green
      set gid 1
      ask neighbors [
        set pcolor green
      set gid 1
      ]
        
   ] ]
end
